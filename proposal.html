Summary: In this project we are attempting to model the behavior of fireworks particles. On top of that, we are hoping to include a user interface that allows us to pick the starting point of the fireworks using mouse clicks, adjust various factors that would affect the behavior such as the initial velocity, the material used to model the fireworks particles, the number of particles that come out of the explosion spot, and etc.

Problem Description:

To model fireworks, we need to simulate a large amount of light objects exploding outwards from a source point. To recreate this, we are planning to simulate many bodyless light emitting particles scattered outwards from a single point. Each particle has the following properties that help us determine its behavior: position, velocity, acceleration, lifetime, and color. The firework simulation is made of different types of particles fulfilling different roles: a trail, exploding particle that creates other particles, and a light particle. Based on its role, a particle will have a different set of rules that determine its behavior and interaction with other particles.

Goals and Deliverables:

Baseline Plan

We plan to not only simulate fireworks but also create a user interface that allows users to create different behaviors or cool effects by manipulating particle properties directly. Some input parameters include number of particles, color, etc. Our demo will create a video of a firework simulation using input setting parameters. To do this our program will create a frame by frame model of the fireworks particles and then create a video from those frames.

To measure quality or performance, we are planning to benchmark speed based on the number of particles we are trying to render. We can potentially add speed up optimization such as parallelization and CUDA and include graphs to display the improvement in speed. With this implemented, we can figure out how much parallelization affects the performance of a rendering system for a huge number of particles.

Aspirational Plan

Interactive mouse click

To improve upon the interface, we can add an interactive UI. This will allow a user to interactively create new fireworks by clicking on the UI, originating a firework at the mouse click point.

Different types of fireworks

In addition to the classic point expansion fireworks, we can also add different types of fireworks. Some examples include fireworks that explode in different shapes (such as an upward cone), fireworks that dim out before falling downwards so the light maintains a circular shape, etc. For these we can establish pre-set parameters for initial velocity and lifetime that can be selected to create different effects. We can also create fireworks with different explosion shapes and patterns.

Smoke effects

With enough time we can also add smoke effects to simulate what a real life fireworks show would look like. To do this we would have to model smoke particles in addition to fireworks particles.

Schedule

Week 1: Setup framework and simulate how particles shoot up before explosion occurs

  • Set up openGL for animation
  • Create module for calculating particle position
  • Use point-sprite particle system
  • Model velocity, acceleration

Week 2: Simulate how explosion effects work

  • Add time to die for particle fade out
  • Model behavior with gravity for individual particles
  • Work on performance for large numbers of particles

Week 3: Create User Interface to manipulate particle properties

  • Add UI for input parameters on a sidebar
  • Pass parameters to particle simulation module

Week 4: Work on our inspirational plan

  • Use openGL to take in mouse input
  • Play around with different settings to create different particle effects
